# FocusFlow 项目交接说明

## 📋 项目概述

**FocusFlow** 是一款专业的工时追踪与项目管理工具，通过自动记录用户在电脑上的活动（应用程序、文件路径），智能分配到对应项目，并提供可视化的工时统计看板。

### 核心价值
- ⏱️ **自动追踪** - 无需手动操作，自动记录工作活动
- 🎯 **智能分配** - 基于规则自动将活动分配到项目
- 📊 **实时统计** - 可视化展示项目工时投入
- 💻 **跨平台** - 支持 macOS、Linux、Windows

---

## 🏗️ 技术架构

### 技术栈
- **GUI 框架**: PySide6 (Qt for Python)
- **数据库**: SQLite3
- **数据处理**: Pandas, Matplotlib
- **系统监控**: 
  - macOS: Quartz (Core Graphics)
  - Windows: psutil (计划)
- **开发语言**: Python 3.8+

### 项目结构
```
FocusFlow/
├── core/                      # 核心模块
│   └── database.py           # 数据库管理、配置管理
├── gui/                       # 图形界面

├── start.sh                  # macOS/Linux 启动脚本
├── start.bat                 # Windows 启动脚本
└── venv/                     # Python 虚拟环境
```

## 🚀 核心功能

### 1. 后台采集引擎 (service_daemon.py)
**职责**: 每秒检测并记录用户活动

**功能**:
- 检测当前活跃窗口 (应用名称、文件路径、窗口标题)
- 检测系统闲置状态 (基于鼠标/键盘活动)
- 将活动记录写入数据库 (activity_log 表)
- 更新实时状态表 (runtime_status)

**关键特性**:
- 使用 `nohup` (macOS/Linux) 或 `start /B` (Windows) 后台运行
- 独立于 GUI，即使关闭界面仍继续采集
- 支持多个 GUI 实例共享同一个采集服务

**运行方式**:
```bash
# 独立运行 (不推荐)
python service_daemon.py

# 推荐：通过启动脚本自动管理
./start.sh  # macOS/Linux
start.bat   # Windows
```

### 2. GUI 主界面 (dashboard_v2.py)
**核心组件**:

#### DashboardV2 类
主窗口，包含所有 UI 组件和业务逻辑

**关键功能模块**:

1. **系统托盘管理 (SystemTrayManager)**
   - 显示/隐藏主界面
   - 显示/隐藏悬浮窗
   - 重启程序
   - 退出程序
   - macOS Dock 图标动态管理 (窗口隐藏时自动隐藏 Dock 图标)

2. **项目树管理**
   - 左侧项目树形结构展示
   - 支持项目展开/折叠状态记忆
   - 支持项目选择状态记忆
   - 显示归档项目开关

3. **实时状态追踪**
   - 每秒刷新当前活动 (应用、文件、项目)
   - 自动匹配项目分配规则
   - 显示累积时间、今日时间、本次连续时间

4. **时间统计面板 (Stats Bar)**
   - 项目统计：累积时间、今日时间
   - 程序统计：累积时间、今日时间、本次连续时间
   - 项目名自动跟随追踪内容变化

5. **数据可视化**
   - 过去 7 天项目时间柱状图
   - 本周项目时间精力分配环形饼图
   - 使用 Matplotlib 渲染

6. **悬浮窗 (FloatingWidget)**
   - 独立可拖拽的紧凑信息窗口
   - 显示当前项目、程序、时间
   - 与主界面数据实时同步

### 3. 项目分配系统

#### 两种分配方式:

1. **手动分配 (file_assignment 表)**
   - 用户右键项目 → "分配文件/窗口"
   - 直接绑定文件路径到项目
   - 优先级最高

2. **自动分配 (project_map 表)**
   - 用户右键项目 → "编辑自动分配规则"
   - 设置路径/窗口标题关键词
   - 模糊匹配 (包含关键词即匹配)
   - 按规则 ID 倒序匹配 (最新规则优先)

#### 匹配优先级:
```
用户手动选择项目 > file_assignment 手动分配 > project_map 自动匹配
```

## 📊 数据库设计

### 核心表结构

#### 1. projects (项目表)
```sql
CREATE TABLE projects (
    id INTEGER PRIMARY KEY,
    project_name TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_archived INTEGER DEFAULT 0,
    parent_id INTEGER  -- 支持父子项目
);
```

#### 2. activity_log (活动记录表)
```sql
CREATE TABLE activity_log (
    id INTEGER PRIMARY KEY,
    timestamp TIMESTAMP,
    duration INTEGER,  -- 秒数
    app_name TEXT,
    file_path TEXT,
    window_title TEXT
);
```

#### 3. file_assignment (手动分配表)
```sql
CREATE TABLE file_assignment (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    file_path TEXT UNIQUE
);
```

#### 4. project_map (自动分配规则表)
```sql
CREATE TABLE project_map (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    rule_path TEXT  -- 关键词，支持模糊匹配
);
```

#### 5. runtime_status (实时状态表)
```sql
CREATE TABLE runtime_status (
    id INTEGER PRIMARY KEY,
    updated_at TIMESTAMP,
    is_idle INTEGER,  -- 是否闲置
    idle_seconds REAL,
    app_name TEXT,
    file_path TEXT
);
```

#### 6. system_config (系统配置表)
```sql
CREATE TABLE system_config (
    key TEXT PRIMARY KEY,
    value TEXT
);
-- 存储配置项：
-- floating_position_x, floating_position_y  (悬浮窗位置)
-- floating_visible  (悬浮窗可见性)
-- idle_threshold  (闲置阈值，秒)
```

#### 7. ignore_list (忽略列表)
```sql
CREATE TABLE ignore_list (
    id INTEGER PRIMARY KEY,
    keyword TEXT UNIQUE  -- 黑名单关键词
);
```

## 🎯 核心功能详解

### 实时追踪逻辑 (每 3 秒刷新)

**位置**: `DashboardV2.refresh_data()`

**执行流程**:

1. **读取实时状态**
   - 从 `runtime_status` 表读取当前是否闲置、应用名称、文件路径
   - 从 `activity_log` 表读取最近的活动记录 (5 秒内)

2. **计算本次连续时长**
   - 如果文件路径未变化，累加 3 秒
   - 如果文件路径变化，重置为 3 秒
   - 闲置时清零

3. **匹配项目**
   - 优先使用用户手动选择的项目
   - 否则查询 `file_assignment` 表
   - 否则查询 `project_map` 表 (模糊匹配)

4. **计算时间统计**
   - 查询当前程序的累积时间和今日时间
   - 查询当前项目的累积时间和今日时间 (支持子项目汇总)

5. **更新 UI**
   - 更新顶部状态栏
   - 更新悬浮窗 (如果可见)
   - 更新 macOS Dock 图标可见性

---

### 闲置检测逻辑

**位置**: `service_daemon.py`

**检测方式** (macOS):
- 使用 Quartz 框架检测系统事件
- 检测键盘、鼠标、滚轮事件
- 支持多种闲置检测模式:
  - `strict`: 严格模式 (任何事件都算活动)
  - `combined`: 综合模式 (默认)

**闲置阈值**:
- 从 `system_config` 表读取 (默认 30 秒)
- 可在界面设置中修改

---

### 项目归档功能

**功能**:
- 归档项目从项目树隐藏 (默认)
- 可勾选"显示归档"查看
- 归档项目仍可分配文件
- 统计数据包含归档项目

**实现**:
```python
# 查询时根据复选框决定是否包含归档项目
if show_archived:
    query = "SELECT * FROM projects"
else:
    query = "SELECT * FROM projects WHERE is_archived = 0"
```

## 🚀 启动流程

### 启动脚本 (start.sh / start.bat)

**执行步骤**:

1. **激活虚拟环境**
   ```bash
   source venv/bin/activate  # macOS/Linux
   call venv\Scripts\activate.bat  # Windows
   ```

2. **检测后台服务**
   ```bash
   pgrep -f "service_daemon.py"  # macOS/Linux
   tasklist /FI "IMAGENAME eq *service_daemon.py*"  # Windows
   ```

3. **启动后台服务** (如果未运行)
   ```bash
   nohup python service_daemon.py > /dev/null 2>&1 &  # macOS/Linux
   start /B python service_daemon.py  # Windows
   ```

4. **启动 GUI 界面**
   ```bash
   python gui/dashboard_v2.py
   ```

## 📝 常见问题

### Q1: 关闭终端后程序停止运行
**解决**: 使用启动脚本 `./start.sh`，后台服务使用 `nohup` 运行

### Q2: 项目名显示"未分配"
**原因**: 文件路径没有分配规则
**解决**: 
- 右键项目 → "分配文件/窗口" 添加手动规则
- 或右键项目 → "编辑自动分配规则" 添加自动匹配关键词

### Q3: 时间统计不准确
**检查**:
1. 后台服务是否在运行 (`pgrep -f "service_daemon.py"`)
2. 数据库是否有活动记录
3. 闲置阈值是否设置过短

### Q4: 多个 GUI 实例
**说明**: 支持多个 GUI 实例共享同一个后台服务
**注意**: 关闭一个 GUI 不会影响后台服务和其他 GUI 实例
