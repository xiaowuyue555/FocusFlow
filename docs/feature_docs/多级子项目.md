# 多级子项目功能说明

## 功能概述

FocusFlow 已实现完整的多级子项目支持，包括树形结构、递归统计、叶子节点归档等功能。

## 核心特性

### 1. 树形项目结构
- 支持无限层级的父子项目关系
- 通过 `parent_id` 字段实现自引用
- 同一父节点下项目名唯一（防止混乱）

### 2. 递归统计
- **单独统计**: 只查看当前项目的工时
- **递归统计**: 包含所有子孙项目的工时汇总
- 父项目自动汇总子项目工时

### 3. 叶子节点归档
- 只有没有子项目的节点才能归档
- 归档后可在设置中选择显示/隐藏
- 恢复归档后项目重新显示

### 4. UI 改进
- 左侧使用 `ttk.Treeview` 显示树形项目列表
- 每个节点显示：总计时、今日时长、子项目数量
- 右键菜单支持：
  - 新建子项目
  - 重命名
  - 归档/恢复
  - 删除项目

## 数据库结构

### projects 表
```sql
CREATE TABLE projects (
    id INTEGER PRIMARY KEY,
    project_name TEXT NOT NULL,
    parent_id INTEGER,  -- NULL 表示根项目
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_archived INTEGER DEFAULT 0,
    FOREIGN KEY (parent_id) REFERENCES projects(id)
);
```

### project_map 表
```sql
CREATE TABLE project_map (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    rule_path TEXT,  -- 路径关键词
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### file_assignment 表
```sql
CREATE TABLE file_assignment (
    id INTEGER PRIMARY KEY,
    file_path TEXT UNIQUE,
    project_id INTEGER,
    assigned_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

### project_archive 表
```sql
CREATE TABLE project_archive (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    archived_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

## API 使用

### 核心函数

```python
from core.project_tree import (
    load_project_tree,      # 加载项目树
    create_project,         # 创建项目
    delete_project,         # 删除项目
    move_project,           # 移动项目
    archive_project,        # 归档项目
    restore_project,        # 恢复项目
    get_project_stats,      # 获取项目统计
    get_all_projects_flat   # 获取所有项目（扁平列表）
)
```

### 示例

```python
# 创建根项目
project_id = create_project("动画项目 A")

# 创建子项目
child_id = create_project("镜头 001", parent_id=project_id)

# 加载树形结构
tree = load_project_tree()
root = tree.get_node(project_id)
for child in root.get_children():
    print(f"子项目：{child.name}")

# 获取统计（包含子项目）
stats = get_project_stats(project_id, include_children=True)
print(f"总计：{stats['total']}秒")
print(f"今日：{stats['today']}秒")
```

## UI 操作指南

### 新建项目
1. 在顶部输入框输入项目名称
2. 可选：输入自动归档关键词
3. 点击"新建项目"按钮

### 新建子项目
1. 在左侧树形列表中右键点击父项目
2. 选择"新建子项目"
3. 输入子项目名称

### 重命名
1. 右键点击项目
2. 选择"重命名"
3. 输入新名称

### 归档/恢复
1. 右键点击项目
2. 选择"归档/恢复"
3. 项目将在列表中隐藏/显示
4. 勾选"显示已归档"可查看所有归档项目

### 删除项目
1. 右键点击项目
2. 选择"删除项目"
3. 确认删除（会同时删除所有子项目）

## 规则匹配

规则绑定到具体项目 ID，支持路径匹配：
- 规则路径包含在文件路径中时自动匹配
- 同一路径命中多个规则时，按规则定义顺序匹配
- 手动分配优先级 > 自动匹配

## 迁移说明

现有数据已自动迁移：
- 所有旧的 `project_name` 映射为顶层节点
- `parent_id` 为 NULL
- 新增项目可设置父子关系

## 注意事项

1. **循环引用检测**: 系统会自动检测并防止循环引用
2. **删除策略**: 删除有子项目的节点会提示错误，需先删除子项目
3. **归档限制**: 只有叶子节点（无子项目）才能归档
4. **同名约束**: 同一父节点下项目名必须唯一
