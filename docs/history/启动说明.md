# FocusFlow 启动说明

## 快速启动

### macOS / Linux
```bash
cd /path/to/FocusFlow
# 方法 1：直接执行脚本（推荐）
bash scripts/start.sh

# 方法 2：赋予执行权限后执行
chmod +x scripts/start.sh
./scripts/start.sh
```

### Windows
双击运行 `scripts/start.bat` 或在命令行执行：
```cmd
scripts\start.bat
```

## 启动流程

启动脚本会自动完成以下操作：

1. **检查后台服务** - 检测 `service_daemon.py` 是否已在运行
2. **启动后台服务** - 如果未运行，使用 `nohup` (macOS/Linux) 或 `start /B` (Windows) 在后台启动
3. **启动 GUI 界面** - 启动 `gui/dashboard_v2.py` 图形界面

## 优势

✅ **只需一个命令** - 无需手动启动两个程序  
✅ **后台服务独立运行** - 关闭终端后服务仍继续运行  
✅ **自动检测** - 不会重复启动后台服务  
✅ **跨平台支持** - macOS、Linux、Windows 均可使用  

## 停止程序

1. **关闭 GUI** - 通过系统托盘菜单选择"退出程序"
2. **停止后台服务**（可选）：
   ```bash
   # macOS/Linux
   pkill -f "service_daemon.py"
   
   # Windows
   taskkill /F /FI "WINDOWTITLE eq *service_daemon.py*"
   ```

## 注意事项

- 首次启动可能需要几秒钟初始化数据库
- 后台服务会在后台持续运行，即使关闭了 GUI
- 多个 GUI 实例可以共享同一个后台服务
- 确保已激活虚拟环境

## 故障排查

### GUI 无法启动
1. 检查虚拟环境是否已激活
2. 确认依赖已安装：`pip install -r requirements.txt`
3. 查看终端错误信息

### 后台服务未运行
1. 手动启动：`python service_daemon.py`
2. 检查是否有权限问题
3. 查看日志输出

### 数据不更新
1. 确认后台服务正在运行
2. 检查数据库文件权限
3. 重启后台服务
