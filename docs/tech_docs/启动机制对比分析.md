# 🖥️ macOS vs Windows 启动机制对比分析

**分析时间**: 2026-03-14  
**目的**: 解答关于双进程启动的疑问

---

## 📋 核心问题

### 问题 1：是否需要启动两个程序？
**答案**: ✅ 是的，但用户无感知

**架构**:
```
用户启动脚本
    ↓
┌─────────────────────────────────┐
│ 1. 后台采集服务 (service_daemon.py) │ ← 静默运行，无界面
│    - 每秒检测应用活动              │
│    - 写入数据库                   │
│    - 后台运行                    │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ 2. GUI 界面 (dashboard_v2.py)      │ ← 用户可见的窗口
│    - 显示统计数据                │
│    - 项目管理系统                │
│    - 系统托盘图标                │
└─────────────────────────────────┘
```

**为什么需要两个进程？**
- **后台服务**: 持续采集数据，即使用户关闭 GUI 也不中断
- **GUI 界面**: 用户交互界面，可以随意开关
- **优势**: 数据采集不受界面开关影响

## 🍎 macOS 启动机制

### 启动流程
```bash
# 1. 用户执行
bash scripts/start.sh

# 2. 脚本检查后台服务
pgrep -f "service_daemon.py"
# 如果未运行 → 启动

# 3. 后台启动（关键）
nohup python service_daemon.py > /dev/null 2>&1 &
# ↑ nohup 让进程忽略挂起信号
# ↑ & 让进程在后台运行
# ↑ > /dev/null 2>&1 丢弃所有输出

# 4. 启动 GUI
python gui/dashboard_v2.py
# ↑ 前台运行，有窗口
```

### 终端窗口行为

**场景 A: 从终端启动**
```bash
$ bash scripts/start.sh
✅ 后台服务已启动
🖥️  启动 FocusFlow 界面...
# 此时：
# - 后台服务：独立运行，不受终端影响
# - GUI：新窗口打开
# - 终端：可以关闭，不影响程序
```

**场景 B: 从 Finder 双击（如果创建 .command 文件）**
```
- 会打开一个新的终端窗口
- 程序运行在该终端中
- 关闭终端 = 关闭 GUI
- 但后台服务继续运行
```

### 关键点

**nohup 的作用**:
```bash
nohup python service_daemon.py &
```
- `nohup` = "no hang up"
- 让进程忽略终端挂起信号
- 即使关闭终端，进程继续运行
- **不会**有终端窗口残留

**实际效果**:
- ✅ 后台服务：无窗口，静默运行
- ✅ GUI：只有 GUI 窗口，无终端
- ✅ 关闭终端：不影响后台服务

## 🪟 Windows 启动机制

### 启动流程
```batch
REM 1. 用户执行（双击或命令行）
python scripts\start.bat

REM 2. 脚本检查后台服务
tasklist /FI "IMAGENAME eq *service_daemon.py*" | find "service_daemon.py"
REM 如果未运行 → 启动

REM 3. 后台启动（关键）
start /B python service_daemon.py
REM ↑ start /B 在后台启动，不创建新窗口

REM 4. 启动 GUI
python gui\dashboard_v2.py
REM ↑ 前台运行，有窗口
```

### 终端窗口行为

**场景 A: 从 CMD/PowerShell 启动**
```cmd
C:\> python scripts\start.bat
✅ 后台服务已启动
🖥️  启动 FocusFlow 界面...
REM 此时：
REM - 后台服务：独立运行
REM - GUI：新窗口打开
REM - CMD 窗口：可以关闭，不影响程序
```

**场景 B: 双击 .bat 文件**
```
- 会打开一个 CMD 窗口
- 执行脚本
- 后台服务启动（无窗口）
- GUI 启动（新窗口）
- CMD 窗口会保持打开（因为 GUI 在前台运行）
```

### ⚠️ Windows 的特殊问题

**问题**: 双击 `start.bat` 会出现 CMD 窗口

**原因**:
```batch
start /B python service_daemon.py  REM 后台服务 - 无窗口
python gui\dashboard_v2.py         REM GUI - 前台运行，但 CMD 窗口不关闭
```

**现象**:
```
┌──────────────────┐
│ CMD 窗口          │ ← 这个窗口会残留
│ C:\> start.bat   │
│ ✅ 后台服务...   │
│ 🖥️  启动...      │
│                  │
│ [窗口保持打开]   │
└──────────────────┘
        ↓
┌──────────────────┐
│ FocusFlow GUI   │ ← 这是 GUI 窗口
│                  │
│ [正常使用]       │
└──────────────────┘
```

## 🔧 解决方案对比

### macOS 方案 ✅ (完美)
```bash
nohup python service_daemon.py > /dev/null 2>&1 &
```
- ✅ 后台服务完全独立
- ✅ 无终端窗口残留
- ✅ GUI 独立窗口
- ✅ 关闭终端不影响

### Windows 方案 ⚠️ (需要优化)

**当前方案**:
```batch
start /B python service_daemon.py
python gui\dashboard_v2.py
```
- ✅ 后台服务无窗口
- ❌ CMD 窗口会残留（双击时）

**优化方案 1**: 使用 VBScript 隐藏窗口
```vbscript
' start_hidden.vbs
Set objShell = CreateObject("WScript.Shell")
objShell.Run "python service_daemon.py", 0, False
```

**优化方案 2**: 使用 Python 启动器
```python
# launcher.pyw (注意 .pyw 扩展名无控制台)
import subprocess
import sys

# 启动后台服务
subprocess.Popen([sys.executable, "service_daemon.py"], 
                 creationflags=subprocess.CREATE_NO_WINDOW)

# 启动 GUI
from gui.dashboard_v2 import DashboardV2
import sys
from PySide6.QtWidgets import QApplication

app = QApplication(sys.argv)
window = DashboardV2()
window.show()
sys.exit(app.exec())
```

**优化方案 3**: 使用 Windows 任务计划程序
- 后台服务注册为任务
- 开机自动启动
- 完全无窗口

## 📊 双进程通信机制

### 问题：两个进程如何联通？

**答案**: 通过 **SQLite 数据库**

```
┌──────────────────┐         ┌──────────────────┐
│  后台采集服务     │         │    GUI 界面       │
│                  │         │                  │
│  检测应用活动     │         │  读取数据库      │
│      ↓           │         │      ↑           │
│  写入数据库      │────────→│  显示统计        │
│  (tracker.db)    │         │                  │
└──────────────────┘         └──────────────────┘
         ↓                           ↓
    SQLite 数据库 (文件)        SQLite 数据库 (文件)
```

### 通信流程

**1. 后台服务写入**:
```python
# service_daemon.py (每秒执行)
conn = get_connection()
conn.execute(
    "INSERT INTO activity_log (timestamp, app_name, file_path, duration) VALUES (?, ?, ?, ?)",
    (datetime.now().isoformat(), app_name, file_path, 1)
)
conn.commit()
```

**2. GUI 读取**:
```python
# dashboard_v2.py (每 3 秒刷新)
conn = get_connection()
data = conn.execute(
    "SELECT SUM(duration) FROM activity_log WHERE date(timestamp) = today"
).fetchone()
```

### 并发控制

**SQLite WAL 模式**:
```python
# 数据库初始化时设置
conn.execute("PRAGMA journal_mode=WAL")
```

**优势**:
- ✅ 支持多进程同时读写
- ✅ 读不阻塞写，写不阻塞读
- ✅ 性能优秀

**实际效果**:
- ✅ 后台服务持续写入
- ✅ GUI 实时读取显示
- ✅ 无冲突，无锁等待
- ✅ 数据一致性保证

## ✅ 回答你的问题

### Q1: Windows 也要启动两个程序吗？
**A**: ✅ 是的，和 macOS 一样，都是双进程架构。

### Q2: 会是后启动的杀死前面的吗？
**A**: ❌ 不会！两个进程独立运行：
- 后台服务：独立运行，不受 GUI 影响
- GUI：可以随时开关
- 关闭 GUI ≠ 关闭后台服务

### Q3: 会不会出现一个终端窗口？
**A**: 
- **当前版本**: ⚠️ 会（双击 .bat 时）
- **优化后**: ✅ 不会（使用 .pyw 或 VBScript）

### Q4: 二者的联通都正常吗？
**A**: ✅ 完全正常！通过 SQLite 数据库通信：
- 后台服务写入数据
- GUI 读取并显示
- SQLite WAL 模式支持并发
- 无冲突，性能优秀

### 立即优化 Windows 启动体验

# launcher.pyw - Windows 专用启动器
import subprocess
import sys
import os

# 获取项目根目录
PROJECT_ROOT = os.path.dirname(os.path.abspath(__file__))

from gui.dashboard_v2 import DashboardV2
from PySide6.QtWidgets import QApplication

app = QApplication(sys.argv)
window = DashboardV2()
window.show()
sys.exit(app.exec())
