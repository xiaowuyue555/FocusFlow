# 🚨 重要：启动方式说明

## ⚠️ 常见错误

**错误方式**（不要这样做）：
```bash
python scripts/start.sh  # ❌ 错误！这是 bash 脚本，不是 Python 脚本
```

会导致类似这样的错误：
```
File "/path/to/scripts/start.sh", line 7
    SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
                       ^
SyntaxError: invalid syntax
```

## ✅ 正确的启动方式

### macOS / Linux

**方法 1：使用 bash 执行（推荐）**
```bash
cd /path/to/FocusFlow
bash scripts/start.sh
```

**方法 2：赋予执行权限**
```bash
cd /path/to/FocusFlow
chmod +x scripts/start.sh
./scripts/start.sh
```

### Windows

```cmd
cd C:\path\to\FocusFlow
python scripts\start.bat
```

或直接双击：
```
scripts\start.bat
```

## 📝 为什么？

- `start.sh` 是 **Bash 脚本**，需要使用 `bash` 或 `sh` 执行
- `start.bat` 是 **Windows 批处理脚本**，由 Windows 自动识别执行
- 不能用 Python 解释器执行 Bash 脚本

## 🔧 启动脚本会做什么？

1. 检查后台服务是否已在运行
2. 如果未运行，启动后台采集服务
3. 启动 GUI 界面
4. 自动激活虚拟环境

## 💡 提示

如果已经赋予了执行权限，可以直接：
```bash
./scripts/start.sh
```

如果没有执行权限，使用：
```bash
bash scripts/start.sh
```
