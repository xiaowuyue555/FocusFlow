# 🚀 启动脚本修复说明

## 📋 问题描述

在整理项目后，用户遇到了启动错误：

```
python scripts/start.sh
File "/path/to/scripts/start.sh", line 7
    SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
                       ^
SyntaxError: invalid syntax
```

## 🔍 原因分析

1. **脚本类型混淆**：`start.sh` 是 Bash 脚本，不是 Python 脚本
2. **路径问题**：脚本切换到 scripts 目录后，相对路径指向错误

## ✅ 解决方案

### 1. 修复启动方式

**错误方式** ❌：
```bash
python scripts/start.sh  # 不要用 Python 执行 Bash 脚本
```

**正确方式** ✅：
```bash
# 方法 1：使用 bash 执行（推荐）
bash scripts/start.sh

# 方法 2：赋予执行权限后执行
chmod +x scripts/start.sh
./scripts/start.sh
```

### 2. 修复路径问题

**更新前**的问题：
```bash
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
cd "$SCRIPT_DIR"  # 切换到了 scripts/ 目录
source venv/bin/activate  # ❌ 路径错误！应该是 ../venv/bin/activate
python gui/dashboard_v2.py  # ❌ 路径错误！应该是 ../gui/dashboard_v2.py
```

**更新后**的解决方案：
```bash
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
PROJECT_ROOT="$( cd "$SCRIPT_DIR/.." && pwd )"  # 获取项目根目录
cd "$PROJECT_ROOT"  # 切换到项目根目录
source venv/bin/activate  # ✅ 正确路径
python gui/dashboard_v2.py  # ✅ 正确路径
```
